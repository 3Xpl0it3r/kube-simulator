# Change: 为优先级包添加单元测试（第一阶段）

## Why
基于代码复杂度和重要性分析，第一阶段优先为核心工具包和Kubernetes资源包添加单元测试。这些包是整个系统的基础，缺乏测试可能导致底层逻辑错误影响整个项目。分阶段实施可以确保测试质量并控制变更风险。

## What Changes
- 为 pkg/util 包添加单元测试（最简单的工具函数包）
- 为 pkg/kuberes 包添加单元测试（Kubernetes资源管理核心包）
- 建立可复用的测试模式和辅助函数
- 确保第一阶段测试覆盖率达到 85%+
- 为后续阶段奠定测试基础

## Impact
- 受影响的代码：pkg/util/ 和 pkg/kuberes/ 目录下所有文件
- 新增文件：*_test.go 测试文件
- 测试框架：基于已有的 Go 标准库 testing 包
- 构建影响：轻微增加测试执行时间
- 开发流程：为底层包建立测试标准

## 分阶段计划
**第一阶段（本变更）**：基础包（pkg/util, pkg/kuberes）
**第二阶段**：中级复杂度包（pkg/cert, pkg/simulator）  
**第三阶段**：复杂集成包（pkg/cluster）
